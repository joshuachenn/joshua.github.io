导出表

动态装载

这一节我们将动态装载一个可执行文件到内存中，目的是在另一个可执行文件中运行被动态装载了PE结构等相关数据的程序。

原理是利用PE结构，将PE结构的头部信息从源可执行文件的FA装载入另一个已经在运行状态的进程虚拟内存中，亦即VA中，除了头部信息之外，我们还要做的是修改IAT项数据，因为要修改IAT数据，我们还要获取导入表相关信息，获取源可执行文件的节表项，通过节表获取需要加载的动态库，然后通过导入表项获取每个节要导入的函数VA，并写入到进程虚拟内存的IAT表项指向的地址下。

PE格式03

发表于 2020-05-08

节表

PE文件中所有节的属性都被定义在节表中，节表由一系列的IMAGE_SECTION_HEADER结构排列而成，每个结构用来描述一个节，结构的排列顺序和它们描述的节在文件中的排列顺序是一致的，全部有效结构的最后以一个空的IMAGE_SECTION_HEADER结构作为结束，所以节表中IMAGE_SECTION_HEADER结构数量等于节的数量+1。节表总是被存放在紧接着PE文件头的地方。

PE格式01

发表于 2020-05-08

介绍

PE格式是Windows可执行文件的文件格式，通常该类型格式以.exe作为后缀，而linux、iOS、安卓等等这些系统平台都有其对应的可执行文件格式，这里我们主要介绍PE格式。

Windows操作系统家族最近增加的Windows NT为开发环境和应用程序本身带来了很大的改变，这之中一个最为重大的当属PE文件格式了。新的PE文件格式主要来自于UNIX操作系统所通用的COFF（Common Object File Format）规范，同时为了保证与旧版本MS-DOS及Windows操作系统的兼容，PE文件格式也保留了MS-DOS中的MZ头部，在16进制下为0x4D5A。
